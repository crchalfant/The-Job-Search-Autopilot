================================================================================
JOB SEARCH AUTOPILOT - SETUP GUIDE
================================================================================

A personal job search automation tool that searches 12 sources daily, rates
every listing with Claude AI, and emails you a ranked digest each morning.

This file is the deep-dive companion to README.md. It covers step-by-step
setup, every customizable filter, and a full set of AI prompts you can paste
directly into Claude or ChatGPT to configure the tool for your situation.


--------------------------------------------------------------------------------
BEFORE YOU START - WHAT YOU NEED
--------------------------------------------------------------------------------

You will need API keys for the following services. Getting all of them takes
about 15 minutes. Detailed signup steps are in Step 2 below.

  Anthropic (Claude AI)     console.anthropic.com         ~$1-5/month
  Brave Search              api.search.brave.com          Free (2,000 req/mo)
  Tavily                    app.tavily.com                Free (1,000 req/mo)
  Adzuna                    developer.adzuna.com          Free
  USAJobs (optional)        developer.usajobs.gov         Free

  Email: You need a Gmail account with an App Password (not your login
  password). Setup instructions are in Step 2 below.

You also need Python 3.10 or later installed.
Download it at: python.org/downloads


--------------------------------------------------------------------------------
FILE STRUCTURE
--------------------------------------------------------------------------------

Your project folder should look like this:

  job-search-profile/
  |-- scripts/
  |   |-- job_radar.py         (main radar - run this daily)
  |   |-- dashboard.py         (Kanban dashboard)
  |   |-- radar_shared.py      (shared utilities - no config needed)
  |   `-- config.py            (YOUR personal config - never commit this)
  |-- config.example.py        (template to copy from)
  |-- .env.example             (API key template to copy from)
  |-- .env                     (YOUR API keys - never commit this)
  |-- requirements.txt
  |-- README.md
  |-- SETUP.txt                (this file)
  |-- LICENSE
  `-- .gitignore


--------------------------------------------------------------------------------
STEP 1 - INSTALL DEPENDENCIES
--------------------------------------------------------------------------------

Open a terminal (Command Prompt or PowerShell on Windows) in your project
folder and run:

  pip install -r requirements.txt

If you get a "pip not found" error, try:

  python -m pip install -r requirements.txt


--------------------------------------------------------------------------------
STEP 2 - GET YOUR API KEYS AND SET THEM UP
--------------------------------------------------------------------------------

Copy the .env.example file to a new file called .env in the same folder:

  Windows:  copy .env.example .env
  Mac/Linux: cp .env.example .env

Then get each key using the instructions below and fill them in.


ANTHROPIC (CLAUDE AI) — required, ~$1-5/month
  Used to rate every job posting. The only paid service.

  1. Go to: console.anthropic.com
  2. Create an account and add a payment method (pay per use, no subscription)
  3. Click "API Keys" in the left sidebar → "Create Key"
  4. Copy the key — it starts with sk-ant-

  ANTHROPIC_API_KEY=sk-ant-your-key-here


BRAVE SEARCH — required, free
  Used to search the web for job postings.

  1. Go to: api.search.brave.com
  2. Click "Get Started" → create a free account
  3. Go to "API Keys" → "Create API Key" → select the Free plan (2,000 req/mo)
  4. Copy the key — it starts with BSA

  BRAVE_API_KEY=BSA-your-key-here


TAVILY — required, free
  A second search engine used as a backup to Brave.

  1. Go to: app.tavily.com
  2. Sign up for a free account
  3. Your API key is shown on the dashboard immediately after signup
  4. Copy the key — it starts with tvly-

  TAVILY_API_KEY=tvly-your-key-here


ADZUNA — required, free
  A job board API that returns structured listings with salary data.
  You need both an App ID and an App Key.

  1. Go to: developer.adzuna.com
  2. Click "Register" → create a free account
  3. Go to "Dashboard" → "API Access Details"
  4. Copy both your App ID and App Key

  ADZUNA_APP_ID=your-app-id
  ADZUNA_APP_KEY=your-app-key


GMAIL APP PASSWORD — required, free
  The radar sends your daily email via Gmail. You need an App Password —
  a 16-character code that lets apps access Gmail without your main password.

  Before you start: 2-Step Verification must be enabled on your Google account.
  Enable it at: myaccount.google.com/security

  1. Go to: myaccount.google.com/apppasswords
     (or: Google Account > Security > 2-Step Verification > App passwords)
  2. Under "App name", type something like: Job Radar
  3. Click "Create"
  4. Copy the 16-character password shown — it looks like: xxxx xxxx xxxx xxxx
     (enter it without spaces)

  GMAIL_ADDRESS=youraddress@gmail.com
  GMAIL_APP_PW=xxxxxxxxxxxxxxxx


USAJOBS — optional, free
  Only needed if you want federal government job listings. Skip if not.

  1. Go to: developer.usajobs.gov
  2. Click "Request an API Key" and fill in the form (approved instantly)
  3. You will receive the key by email
  4. Set both variables — the key and the email address you registered with

  USAJOBS_API_KEY=your-key-here
  USAJOBS_EMAIL=youraddress@gmail.com


Save the .env file when done. Never commit it to GitHub — the .gitignore
already protects it.


--------------------------------------------------------------------------------
STEP 3 - SET UP YOUR CONFIG FILE
--------------------------------------------------------------------------------

Copy the config.example.py file to scripts/config.py:

  Windows:  copy config.example.py scripts\config.py
  Mac/Linux: cp config.example.py scripts/config.py

Now open scripts/config.py in a text editor and customize it for yourself.
See the "CUSTOMIZING YOUR CONFIG WITH AI" section below for the easiest way
to do this — you can have Claude or ChatGPT fill in most of it for you.

The key things to update:

  PROFILE           Your current role, target roles, and career summary
  MIN_SALARY        Your minimum acceptable salary (number, e.g. 120000 for $120K)
  LOCAL_METRO_TERMS Your city and surrounding metro area names
  COMPANIES         ATS company slugs for employers you want to track directly
  VACATION_START    Set to a past date if you are not on vacation
  VACATION_END      Set to a past date if you are not on vacation


--------------------------------------------------------------------------------
STEP 4 - RUN THE RADAR
--------------------------------------------------------------------------------

From your project root folder, run:

  python scripts/job_radar.py --run

The first run takes 5-10 minutes. You will see each source being searched
in the console. When it finishes, check your email inbox for the digest.

Add --verbose to see source latencies and filter details:

  python scripts/job_radar.py --run --verbose


--------------------------------------------------------------------------------
STEP 5 - START THE DASHBOARD
--------------------------------------------------------------------------------

Run the dashboard to see your pipeline as a Kanban board:

  python scripts/dashboard.py

Then open http://localhost:5000 in your browser.

The dashboard has three tabs:
  Board     - Your active pipeline (Reviewing, Applied, Interviewing)
  Skipped   - Everything the radar filtered out (good for spot-checking)
  Health    - Run history, source counts, and filter breakdowns


--------------------------------------------------------------------------------
STEP 6 - SCHEDULE DAILY RUNS
--------------------------------------------------------------------------------

WINDOWS - Task Scheduler:
  1. Open Task Scheduler (search for it in the Start menu)
  2. Click "Create Basic Task"
  3. Name: Job Search Autopilot
  4. Trigger: Daily, at 7:00 AM (or whenever you want your morning email)
  5. Action: Start a program
  6. Program: python
  7. Arguments: scripts\job_radar.py --run
  8. Start in: C:\path\to\job-search-profile
  9. Click Finish

MAC/LINUX - Cron job:
  Open a terminal and type: crontab -e
  Add this line (runs at 7am on weekdays):

    0 7 * * 1-5 cd /path/to/job-search-profile && python scripts/job_radar.py --run

  Save and exit.


================================================================================
WHAT FILTERS CAN YOU CUSTOMIZE?
================================================================================

The radar has several filtering layers you can tune. Here is what each one
does and where to find it:

SALARY FLOOR
  File: scripts/config.py — variable: MIN_SALARY
  Set this to your minimum acceptable annual salary as a plain number with no
  commas or dollar sign (e.g. 120000 for $120K). Jobs with a confirmed salary
  range entirely below this are filtered out. Jobs with no salary listed always
  pass through — the radar never drops a job just because salary is unlisted.

LOCATION FILTER — CITIES TO BLOCK
  File: scripts/job_radar.py — variables: _URL_CITY_RE and _LOC_CITY_RE
  Two regex lists that detect when a job requires onsite attendance in a city
  outside your metro. They come pre-loaded with major US metros and
  international cities. To customize:
    - Remove any cities that are within your commutable area
    - Add any cities you want to block that are not listed
  Jobs with "remote" anywhere in the title or description always pass through
  regardless of what these lists contain.

YOUR LOCAL METRO AREA
  File: scripts/config.py — variable: LOCAL_METRO_TERMS
  A set of city and suburb names that define your local commutable area. The
  radar uses this to identify local hybrid/onsite roles and show them in a
  separate section of your daily email. Replace the example values with your
  own city and nearby suburbs.

HARD DISQUALIFIERS (auto-Skip before Claude)
  File: scripts/job_radar.py — list: HARD_DISQUALIFIERS
  Phrases that automatically Skip a job before it reaches Claude. Common
  entries include staffing agency language ("on behalf of our client"),
  contract role signals ("C2C", "corp to corp"), and domain-specific terms
  you never want to see. Add or remove entries to match your situation.

STAFFING AGENCY BLOCKLIST
  File: scripts/job_radar.py — dict: _COMPANY_PREFILTER
  Known staffing and recruiting firms whose listings are skipped automatically.
  Add the name of any agency you keep seeing that you want to block.

JOB TITLE FILTER
  File: scripts/job_radar.py — list: TARGET_TITLES (inside search_ats_companies)
  Controls which job titles are considered relevant when checking ATS boards.
  Add or remove titles to match your target roles.

CLAUDE RATING PROMPT
  File: scripts/job_radar.py — search for: "Perfect Fit" = 90-100%
  The most powerful customization. The tier definitions and hard disqualifier
  rules are plain English — edit them to describe your own situation. See
  Prompt 4 in the AI CUSTOMIZATION section below for a ready-made prompt to
  help you do this with Claude or ChatGPT.


================================================================================
CUSTOMIZING YOUR CONFIG WITH AI — VIBE CODING
================================================================================

This tool is designed to be customized to your specific job search. The
filters, search queries, company list, and rating logic are all plain text
or simple Python lists — readable and editable even if you are not a developer.

The easiest way to personalize it is to upload the files directly into
Claude (claude.ai) or ChatGPT (chatgpt.com) and describe what you want.
You do not need to understand every line of code. Just tell the AI what to
change and it handles the syntax. This is vibe coding — you drive with plain
English, the AI does the editing.

HOW TO DO IT
  1. Go to claude.ai or chatgpt.com
  2. Upload job_radar.py and config.example.py as file attachments
  3. Paste one of the prompts below (fill in your details)
  4. Copy the updated code back into your files and run it

You can do multiple rounds — start with the basics (profile, city, titles)
and iterate from there. The AI can also help you debug errors, add new
features, or explain what any part of the code does.

THINGS YOU WILL NEED AI TO HELP YOU UPDATE

  config.py:
    PROFILE              Rewrite with your background, target roles,
                         strengths, and hard constraints
    LOCAL_METRO_TERMS    Replace placeholders with your city and suburbs
    MIN_SALARY           Your salary floor as a plain number
    All query lists      ADZUNA_QUERIES, BRAVE_QUERIES, TAVILY_QUERIES,
                         LI_REMOTE_QUERIES, LI_LOCAL_QUERIES,
                         HIMALAYAS_QUERIES, REMOTIVE_QUERIES, JOBICY_QUERIES
                         Replace [Your Role] and [Your Industry] placeholders

  job_radar.py:
    TARGET_TITLES        Keywords matching your target job titles — critical,
                         anything that does not match is dropped immediately
    HARD_DISQUALIFIERS   Phrases for things you never want to see
    SALARY_FLOOR_EXEMPT  Employers you trust to pay above your floor
    _URL_CITY_RE         Remove cities near you from the block list
    _LOC_CITY_RE         Same — location field version of the city filter
    Claude rating prompt Rewrite tier definitions for your situation

PROMPT 1 - SET UP YOUR CANDIDATE PROFILE
--------------------------------------------------------------------------------

Use this to generate your PROFILE block. This is what Claude reads when rating
each job — the better your profile, the better the ratings.

Paste this into Claude or ChatGPT:

---
I am setting up a job search automation tool and need help writing my candidate
profile block. Here is my background:

Current role: [Your current title] at [Your current company], [start date] - present
Previous roles: [List your 2-3 most recent roles with titles and companies]
Key achievements: [2-3 specific metrics or accomplishments, e.g. "grew X from Y to Z"]
Target roles: [List the job titles you are looking for]
Target industries: [e.g. healthcare tech, SaaS, e-commerce]
Target salary: $[amount]+
Location: Remote (nationwide) OR on-site/hybrid in [Your city] area only
Hard no's: [Things you definitely do not want, e.g. people management, crypto]
Core strengths: [3-5 things you are genuinely great at]

Please write a PROFILE string for my config.py file. Keep it factual and
concise — 8 to 12 lines. Format it exactly like this:

PROFILE = """
[profile content here]
"""
---

PROMPT 3 - BUILD YOUR ATS COMPANY LIST
--------------------------------------------------------------------------------

Use this to get help building the COMPANIES list. This is the most powerful
feature — the radar checks job boards directly at companies you care about,
so you see roles before they show up on aggregator sites.

Paste this into Claude or ChatGPT:

---
I am building a list of companies to monitor directly via their ATS job boards
(Greenhouse, Lever, and Ashby). The tool tries all three APIs automatically
for each slug.

My target industries are: [e.g. healthcare tech, SaaS, e-commerce]

My target roles are: [e.g. Senior Product Manager, Product Owner, Business Analyst]

Please suggest 40-60 companies in these industries that are known to use
Greenhouse, Lever, or Ashby for hiring, and give me their likely ATS slug.

The slug is usually just the company name lowercased, sometimes with hyphens
for spaces. For example:
  Stripe -> "stripe"
  Cash App -> "cash-app"
  Plaid -> "plaid"

Format your answer as a Python list I can paste into my config.py, with
comments grouping companies by category. Like this:

COMPANIES = [
    # Category name
    "company-one",
    "company-two",
]
---

PROMPT 4 - TUNE THE CLAUDE RATING PROMPT
--------------------------------------------------------------------------------

The rating prompt is what Claude uses to decide if a job is Perfect Fit,
Good Fit, Worth a Look, or Skip. You can tune it to match your specific
situation. Find it in job_radar.py by searching for the text: 90-100%

Paste this into Claude or ChatGPT:

---
I have a job search automation tool that uses Claude AI to rate job listings.
The rating prompt currently defines four tiers: Perfect Fit, Good Fit,
Worth a Look, and Skip.

Here is my background: [paste your PROFILE here]

Here is my situation: [describe anything specific, e.g. "I keep getting rated
Skip for roles at healthcare companies even though I have relevant experience",
or "I want to make sure SaaS roles are weighted higher than other industries"]

Here is the current relevant section of my rating prompt:
[paste the lines from job_radar.py starting with "Perfect Fit  = 90-100%"
through the end of the HARD DISQUALIFIERS section]

Please suggest specific changes to improve the accuracy of ratings for my
situation. Show me the exact text to replace, and what to replace it with.
---

================================================================================
FREQUENTLY ASKED QUESTIONS
================================================================================

Q: The radar ran but I got no email - what happened?
A: Check your .env file. The most common cause is an incorrect GMAIL_APP_PW.
   Make sure you used a Gmail App Password (the xxxx-xxxx-xxxx-xxxx format),
   not your regular Gmail login password. Also check your spam folder.

Q: A source is showing 0 results - is something broken?
A: The radar will warn you in the email subject line if an expected source
   returns zero results. Check your API keys in .env for that source. LinkedIn
   rate-limits aggressively so occasional zeros there are normal.

Q: Jobs from companies I rejected keep coming back - why?
A: Reject jobs using the trash icon on the dashboard board tab, not just the
   skipped tab. Rejected jobs are permanently hidden. Dismissed jobs on the
   skipped tab just hide them from that view but they can resurface after 60 days.

Q: How do I add a company to track?
A: Find their ATS slug (see Prompt 5 above) and add it to the COMPANIES list
   in scripts/config.py. The radar will start picking up their jobs on the
   next run.

Q: Can I run it more than once a day?
A: Yes. Each run creates its own output file with a timestamp, so multiple
   daily runs are safe and all results show up on the dashboard. The folder
   auto-prunes to keep only the last 3 runs.

Q: How do I change the salary floor?
A: Change MIN_SALARY in scripts/config.py. It is a number without commas
   or dollar signs (e.g. 120000 for $120K).

Q: The dashboard shows old jobs that I have already seen - is this normal?
A: Yes, the dashboard shows the last 3 runs worth of jobs. Jobs you have
   rejected will be permanently hidden. New jobs from the latest run will
   appear at the top of the Reviewing column sorted by tier.

Q: I am not a developer. Can I still use this?
A: Yes. The setup steps above are the only technical part. Once it is running,
   everything else — customizing filters, tuning the rating prompt, adding
   companies, debugging problems — can be done by pasting prompts into Claude
   or ChatGPT and copying the output back. You do not need to write any code.
